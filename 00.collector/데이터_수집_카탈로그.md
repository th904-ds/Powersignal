# Powersignal 데이터 수집 카탈로그

> 생성일 **2026-06-06** · 인증키 1개(`KPX_SERVICE_KEY`) · 저장 `data/processed/<key>/<단위>.parquet`
> "현재 수집행수"는 이날 실제 1회 수집한 결과. (그대로 노션에 붙여넣어 사용)

## ✅ 결론: 10종 전부 수집 확인됨
10개 데이터셋 모두 실제 API 호출로 **정상 데이터 반환을 확인**했습니다. 운영상 주의 2가지만 기억하세요.
- **solar_wind는 초대형**(파일 1개 ≈ 74만 행 / 약 75콜). 하루 호출예산(90콜)을 거의 다 쓰므로 **다른 데이터셋과 분리해 며칠에 걸쳐 파일 단위로** 수집해야 합니다.
- **gen_by_source는 호출 시점 1행(현재값)** 만 반환 → 시계열 누적은 `daily` 반복 실행으로 쌓입니다.

## 2. 컬럼 상세

### 1) `smp_dayahead` — 계통한계가격+수요예측
| 컬럼 | 의미 |
|---|---|
| `date` | 거래일(YYYYMMDD) |
| `hour` | 시간(1~24) |
| `areaName` | 지역(육지/제주) |
| `smp` | 계통한계가격(원/kWh) |
| `jlfd` `slfd` `mlfd` | 수요예측 값(부하예측 계열) |
| `rn` | 행번호 |

### 2) `power_supply_today` — 오늘 전력수급현황(5분)
| 컬럼 | 의미 |
|---|---|
| `baseDatetime` | 기준시각(YYYYMMDDHHMMSS, 5분 간격) |
| `suppAbility` | 공급능력 |
| `currPwrTot` | 현재부하(현재 수요) |
| `forecastLoad` | 예측부하 |
| `suppReservePwr` / `suppReserveRate` | 공급예비력 / 공급예비율 |
| `operReservePwr` / `operReserveRate` | 운영예비력 / 운영예비율 |

### 3) `gen_by_source` — 발전원별 발전량(현재 5분)
| 컬럼 | 의미 |
|---|---|
| `baseDateTime` | 기준시각 |
| `fuelPwr1`~`fuelPwr10` | 연료원별 발전량(원자력·석탄·LNG·신재생 등 10종) |
| `fuelPwrTot` | 총발전량 |
| `pEsmw` `bEmsw` | 추가 계측값(예비/기타) |

### 4) `solar_wind_by_region` — 지역별 시간별 태양광·풍력
| 컬럼 | 의미 |
|---|---|

| `시간` | 시간(0~23) |

| `전력거래량` | 시간대 발전(거래)량 |

| 컬럼 | 의미 |

| `day` | 연료비용 월(YYYYMM) |

| `untpc` | 단가 |

### 6) `smp_decision_count` — 연료원별 SMP 결정횟수(일별)
| 컬럼 | 의미 |
|---|---|
| `tradeDay` | 거래일(YYYYMMDD) |

### 7) `dr_economic` — 경제성DR 거래실적
| 컬럼 | 의미 |
|---|---|

| `qty` | 입찰량(추정) |

### 8) `dr_reliability` — 신뢰성DR 거래실적 *(센티넬 행 자동제거 적용)*
| 컬럼 | 의미 |

| 컬럼 | 의미 |

### 10) `dr_plus` — 플러스DR 입낙찰·이행량
| 컬럼 | 의미 |
|---|---|

## 3. 운영 주의사항
1. **호출예산**: 개발계정 하루 100콜(코드 안전마진 90). 한 번의 `run.py` 실행은 90콜에서 깔끔히 멈추고, 재실행 시 체크포인트(parquet 존재)로 이어받음.
2. **solar_wind 분리 수집 권장**: 파일 1개가 ~75콜이라 다른 데이터셋과 같은 실행에 넣으면 예산을 독식함. 예) `python run.py backfill --start 2024-01-01 --end 2024-01-01 --datasets solar_wind_by_region` 를 며칠 반복 → 파일 10개 순차 완성.
3. **SMP·DR 전체이력 일괄**: `smp_dayahead`, `dr_*` 4종은 1콜로 전체이력을 페이지 반환 → snapshot 모드로 매 실행 시점 스냅샷 1장 저장(시점별 보관).
4. **gen_by_source**: 호출 시점 1행만 옴 → `python run.py daily` 를 스케줄러로 5분/주기 누적.
5. **dr_reliability 센티넬**: 원본에 `tradeDay`가 `9999xxxx`·`3000xxxx`인 플레이스홀더 행이 섞여 있어, `date_sanity_col` 설정으로 수집 시 자동 제거(이번 확인분 58행 제거).

## 4. 전체 백필 권장 순서
```bash
# 1) 가벼운 것 먼저 (SMP·DR·연료비·결정횟수) — 한 번에 끝남
python run.py backfill --start 2014-01-01 --end 2026-06-06 \
  --datasets smp_dayahead,power_supply_today,gen_by_source,monthly_fuel_cost,smp_decision_count,dr_economic,dr_reliability,dr_voluntary,dr_plus

# 2) solar_wind 만 따로, 예산 소진되면 재실행으로 파일 1~2개씩 채움
python run.py backfill --start 2024-01-01 --end 2024-01-01 --datasets solar_wind_by_region
#   (budget_exceeded 뜨면 같은 명령 재실행 → 다음 파일 이어받기)

# 3) 매일 증분
python run.py daily
```
